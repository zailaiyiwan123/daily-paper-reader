## 摘要
识别告诉代理图像中有什么；个人记忆影响接下来值得查询什么。在以相机为先的设置中，用户只能发送一张图像，因此代理必须形成查询。我们研究个人视觉记忆是否能改善代理侧的工具选择和工具参数，从而产生更符合用户意图的多工具查询。该设计使用三层个人视觉记忆（档案、短期焦点、观察），在每一轮加载以调节LLM工具调用循环，在相机优先输入下工作，并包括冲突感知的回写，旨在为后续捕获刷新用户模型。在800张图像与为受控消融构建的合成记忆块配对时，移除完整的三层记忆块会使工具查询相关性绝对降低0.47分（5分制中从4.21降至3.74；相对降低11.2%），端到端效用绝对降低0.082（从0.842降至0.760；相对降低9.7%）。这些结果衡量了在仅图像输入下，使用固定合成块对工具策略的记忆条件化，而非来自真实用户历史的多会话回写。

## Abstract
Recognition tells an agent what is in an image; personal memory affects what is worth looking up next. In a camera-first setting the user can send only an image, so the agent must form the lookups. We study whether personal visual memory improves agent-side tool choice and tool arguments, and thereby more user-aligned multi-tool lookups. The design uses a three-layer personal visual memory (profile, short-term focus, observations) that is loaded on each turn to condition an LLM tool-calling loop under camera-first intake, and includes conflict-aware write-back intended to refresh the user model for later captures. On 800 images paired with synthetic memory blocks constructed for controlled ablation, removing the full three-layer memory block reduces tool-query relevance by 0.47 points absolute (4.21 -> 3.74 on a 5-point scale; 11.2% relative) and end-to-end utility by 0.082 absolute (0.842 -> 0.760; 9.7% relative). These results measure memory conditioning of tool policy under image-only intake with fixed synthetic blocks, not multi-session write-back from live user histories.

### 论文详细总结

#### 1. 核心问题与整体含义（研究动机与背景）

- **核心问题**：在“以相机为先”的视觉代理（Agent）设置中，用户只发送一张图片，没有额外的文字查询。代理需要自主决定“接下来查什么”。论文的核心研究问题是：**个人视觉记忆能否改善代理的工具（API）选择和工具参数设置，从而产生更符合用户个性化需求的查询结果？**
- **背景与现实瓶颈**：现有的视觉系统（如图像识别模型）能很好地回答“图像里有什么”，但无法判断“对特定的用户来说，什么是值得深入探究的”。比如，一位时尚设计师和一位软件工程师看到同一件衣服，他们想查询的信息（流行趋势 vs. 价格面料）完全不同。现有的多模态搜索代理通常依赖明确的查询（如“图像-问题对”），不适用于仅凭图像输入、无文字查询的场景。
- **整体含义**：论文提出了一个连接点，即**个人记忆 → 更好的工具调用 → 更符合用户的多工具查询**。这为构建能主动理解用户意图、并提供个性化服务的视觉代理提供了关键设计思路。

#### 2. 方法论：核心思想、关键技术与流程

- **核心思想**：将个人记忆作为一种**条件信号**注入到大型语言模型（LLM）的工具调用循环中，使LLM在选择工具和构造查询参数时，不仅基于图像内容，还基于对用户的了解，从而实现个性化。
- **关键技术细节**：
    - **三层个人视觉记忆（Memory Substrate）**：
        1.  **档案（Profile）**：长期稳定的用户画像（约800字符）。包括知识水平、兴趣、偏好等。由一个独立的“档案代理”根据用户操作（非模型响应）异步生成。
        2.  **短期焦点（Short-Term Focus）**：用户当前的兴趣领域（约200字符）。例如“正在研究中世纪家具和建筑照明”。每次交互后更新。
        3.  **观察（Observations）**：细粒度的、可检索的用户偏好、约束、比较等语句（50-150条）。通过主题标签和关键词双路径检索，与当前视觉上下文匹配后注入。
    - **内循环（Inner Loop）**：每轮交互的核心过程。
        1.  **记忆加载**：与图像视觉上下文并行异步加载，不阻塞首Token延迟。
        2.  **记忆注入**：以结构化XML块的形式将三层记忆注入到LLM的上下文中，作为工具调用的校准信号。
        3.  **记忆条件化的工具调用**：
            **工具选择**：代理根据记忆决定调用哪些工具（如网络搜索、图片搜索、价格查询、评论等）。
            **参数生成**：代理根据记忆调整查询参数，如深度（专业术语 vs. 通俗语言）、角度（价格 vs. 历史）、约束（避免用户不喜欢的方面）。
        4.  **多步工具使用**：代理可以执行多步工具调用，根据上一步结果精确化下一步的参数，实现自适应检索深度。
    - **外循环/写回（Outer Write-Back）**：一种设计而非评估。
        - 在每个交互回合后，以“即忘即取”的后台任务方式运行，更新记忆以备后续使用。
        - 从交互中提取观察候选，并通过冲突感知操作（增加、更新、删除、无操作）来维护一个简洁、无矛盾的记忆存储。
- **算法流程**：输入图像 → (并行) 获取视觉上下文 + 加载个人记忆 → LLM在记忆条件下决定调用的工具和参数 → 执行工具 → (可选) 再次调用工具或开始合成最终回答 → 输出结果 → (后台) 写回并更新记忆。

#### 6. 主要结论与发现

- **记忆对工具调用有显著影响**：移除完整的三层记忆块，导致：
    - **工具查询相关性**下降**0.47**分（从4.21降至3.74，相对下降11.2%）。
    - **端到端效用**下降**0.082**（从0.842降至0.760，相对下降9.7%）。
- **不同记忆层的影响**：部分消融实验表明，去除“档案”比去除“观察”对效用影响更大（-0.049 vs. -0.031），说明长期稳定的用户画像比短期细节观察更重要。

- **管道组件重要性排序**：从管道消融来看，影响程度从大到小依次是：LLM合成（-0.111）、工具循环（-0.094）、个人记忆（-0.082）、视觉上下文（-0.059）。这说明虽然记忆很重要，但模型的生成能力和工具规划能力是关键。
- **定性示例**：无记忆时，代理对一张手表图片进行通用识别和价格查询。激活收藏家记忆后，代理会进行特定型号对比和转售价值查询，工具调用更具深度和针对性。
