## 摘要
移动和可穿戴设备上的个人AI助手通过视觉和音频流持续感知用户的日常生活。然而，回答关于过去经历的查询需要能够持续积累、组织和检索长期经验的轻量级多模态记忆，这仍然具有挑战性。为了解决这一挑战，我们提出了LightMem-Ego，一个用于日常生活辅助的轻量级流式多模态记忆系统。该系统持续捕获第一人称的视觉和音频流，将它们对齐到共享时间线上，并组织成由当前记忆、短期记忆和长期记忆组成的层次化记忆。给定用户查询，LightMem-Ego动态地将检索路由到适当的记忆层级，并基于多模态证据生成答案。该演示可以在智能手机和AI眼镜上部署，支持物体查找、对话回忆、生活总结、日常发现和个性化辅助。代码可在https://github.com/zjunlp/LightMem-Ego获取。

## Abstract
Personal AI assistants on mobile and wearable devices continuously perceive users' daily lives through visual and audio streams. However, answering queries about past experiences requires lightweight multimodal memory that can continuously accumulate, organize, and retrieve long-term experiences, which remains challenging. To address this challenge, we present LightMem-Ego, a lightweight streaming multimodal memory system for everyday-life assistance. The system continuously captures egocentric visual and audio streams, aligns them on a shared timeline, and organizes them into a hierarchical memory consisting of current, short-term, and long-term memory. Given a user query, LightMem-Ego dynamically routes retrieval to the appropriate memory level and generates answers grounded in multimodal evidence. The demonstration can be deployed on smartphones and AI glasses, supporting object finding, conversation recall, life summarization, routine discovery, and personalized assistance. Code is available at https://github.com/zjunlp/LightMem-Ego.

## 2. 方法论：核心思想、关键技术细节、算法流程
### 核心思想
- **LightMem-Ego** 是一个可部署的流式多模态记忆系统，连接轻量级客户端（手机或AI眼镜）与后端，后端将自中心视觉-音频流摄入、分割为事件，并组织为三级记忆层次结构：**当前记忆**（M_cur）、**短期记忆**（M_st）、**长期记忆**（M_lt，包含情景记忆M_epi和语义记忆M_sem）。
- 回答用户查询时，**记忆路由器**根据时间范围和语义意图选择证据，生成基于多模态证据的回答。

### 关键技术细节
1. **多模态生活捕获**：将日常经验表示为时间有序的多模态流 X = {x_t}_{t=1}^T，其中 x_t = (v_t, a_t, m_t) 分别表示视觉、音频和元数据。客户端轻量级采集，上传低帧率帧和短音频块，避免端侧重推理。
2. **事件分割**：增量式将流分割为短期事件段 e_i，基于时间连续性和跨帧变化信号，无需对每一帧进行语义解析。每个事件存储时间跨度、代表性帧、临时视觉描述和对齐/待处理的音频上下文。
3. **层次化记忆**：
   - **当前记忆**：轻量工作记忆，存储最近的多模态观测和活跃事件状态，用于即时场景响应。
   - **短期记忆**：存储近期事件段，附带视觉证据、描述和转录上下文。
   - **长期记忆**：通过异步合并将稳定事件转化为情景记忆（事件中心）和语义记忆（规律、偏好、关系）。
4. **边缘效率**：客户端仅负责采样、压缩、时间戳标记和上传；后端在线路径更新当前记忆和构建短期记忆，代价较高的操作（ASR回填、事件细化、索引、语义提取）异步执行。查询路由器选择最便宜且足够的源：当前场景查询用M_cur，近期回忆用M_st，回顾或例行问题用M_lt。
5. **经验检索与问答**：给定查询q，路由器产生证据集R(q)，融合异质信号（观测、事件记录、语义摘要、时间戳、帧、转录片段）形成紧凑证据视图E_q，生成答案ŷ = f(q, E_q)。

### 算法流程（文字说明）
- 在线阶段：客户端持续上传帧和音频块 → 后端维护当前记忆（滚动缓冲区）并构建短期事件（通过时间连续性和帧变化） → 短期事件稳定后异步合并到长期记忆（包括事件索引和语义提取）。
