## 摘要
尽管大型语言模型（LLMs）因其强大的推理能力和广泛的世界知识最近在推荐系统中受到关注，但先前基于LLM的智能体存在幻觉和上下文长度限制的问题，因此不适合全排名推荐任务。为了通过架构设计而非修改LLM本身来规避这些限制，我们提出了一种基于智能体的推荐框架——基于自主语言智能体的记忆驱动个性化推荐工具学习（PRTA），其中LLM作为中央规划器，与多个推荐模型作为工具进行交互。基于LLM的智能体负责高级推理和个性化工具选择，而传统推荐模型则利用其在行为模式建模方面的可扩展性执行全排名评分。为支持个性化工具选择，我们设计了反思机制，使智能体能够根据用户画像和候选排名列表为每个用户评估和比较工具。在三个公共数据集上的大量实验表明，PRTA在提升全排名推荐性能方面优于传统推荐和基于LLM的基线方法。

## Abstract
Although large language models (LLMs) have recently gained traction in recommender systems due to their strong reasoning capabilities and extensive world knowledge, previous LLM-based agents suffer from hallucination and context-length limitations, and thus are not suitable for full-ranking recommendation tasks. To circumvent these limitations through architectural design rather than modifying the LLM itself, we propose an agent-based recommendation framework, memory-based $\textbf{P}$ersonalized $\textbf{R}$ecommendation $\textbf{T}$ool learning via autonomous language $\textbf{A}$gents (PRTA), in which an LLM acts as a central planner interacting with multiple recommendation models as tools. The LLM-based agent is responsible for high-level reasoning and personalized tool selection, while traditional recommendation models perform full-ranking scoring, leveraging their scalability in modeling behavioral patterns. To support personalized tool selection, we design reflection mechanisms that enable the agent to evaluate and compare tools for each user based on user profiles and candidate ranked lists. Extensive experiments across three public datasets demonstrate the superiority of \modelname over traditional recommendation and LLM-based baselines in improving full-ranking recommendation performance.

## 2. 论文提出的方法论：核心思想、关键技术细节

*   **核心思想**：将行为建模与语义推理解耦。LLM智能体负责基于用户画像和物品文本描述进行“元推理”（选择哪个推荐工具对当前用户更重要），而传统推荐模型（LightGCN、SASRec、SimpleX）负责实际的全物品评分。
*   **关键技术细节**：
    *   **用户画像生成**：用LLM根据用户历史物品描述生成用户画像（$y_{\text{prof}}$）。
    *   **个性化工具记忆（PT Memory）**：为每个用户维护一个工具权重向量 $\mathbf{m} \in \mathbb{R}^{1 \times d}$（$d$为工具数），通过加权聚合各推荐模型的评分列表 $\hat{\mathbf{r}}_t$，得到最终排名 $\hat{\mathbf{r}} = \sum m_t \hat{\mathbf{r}}_t$。
    *   **智能体反射机制**：通过三种方式迭代优化每个用户的PT Memory：
        *   **局部工具评估**：对每个工具，让LLM判断其Top-$k_{\text{eval}}$结果与用户最近交互的语义相关性（$-1,0,1$），并更新权重 $\mathbf{m}_t \leftarrow \mathbf{m}_t + \alpha \cdot y_{\text{eval}}^t$。
        *   **全局工具比较**：让LLM在所有工具输出中选出与用户历史最相关的一个工具，并更新权重 $\mathbf{m}_t \leftarrow \mathbf{m}_t + \beta \cdot y_{\text{comp}}^t$。
        *   **排名比较**：基于用户最近交互物品在各工具中的实际排名，通过倒数排名归一化后直接更新权重，避免LLM文本判断的偏差。
    *   **推理与上下文重排序**：在聚合排名后，用LLM对Top-$k_{\text{rank}}'$个候选物品进行轻量级语义重排序，得到最终排名列表。
*   **算法流程（文字说明）**：
    1.  **训练阶段**：为每个用户独立进行。
        *   生成用户画像。
        *   对每个训练epoch：
            *   利用各推荐工具生成全量评分列表。
            *   通过局部工具评估、全局工具比较、排名比较三种方式，逐用户更新PT Memory中的工具权重。
    2.  **推理阶段**：
        *   根据更新后的PT Memory加权聚合各工具评分，得到初步排名。
        *   利用LLM和用户画像对Top候选物品进行语义重排序，输出最终结果。

*   **个性化与可解释性**：PT Memory为每个用户学习不同工具权重，不仅能提升准确性，还天然提供了工具层面可解释性（知道对某用户哪种推荐器更重要）。

*   **轻量化重排序**：只在Top-N候选上进行LLM重排序，计算成本可控，且效果明显。
