## 摘要
生成式检索已成为工业搜索、推荐和广告系统中越来越广泛采用的范式，可带来显著的线上收益。现有工作大多将用户行为序列与大语言模型（LLM）结合以建模用户偏好。在实践中，特征工程对模型效果仍然至关重要，但其复杂性拖慢了离线迭代速度，使得线上部署沉重且难以复用，同时面临严格的线上延迟预算约束。根本原因在于特征处理逻辑与模型架构之间的紧耦合，每次特征变更都会触及训练和服务代码，且难以跨场景复用。为打破这一耦合，我们提出提示生成（PG）框架，这是一种高级分词器与配置驱动的框架，通过两个声明式JSON文件将特征处理逻辑与模型架构解耦，这两个文件作为离线训练和线上服务的单一事实来源，确保两阶段特征一致性。PG将特征组织为四种类型，并提供三种可组合的处理组件来组装和压缩异构特征，从而在三个层面实现加速：（1）快速训练迭代：特征实验只需更改配置，内置针对超长序列的令牌压缩；（2）快速部署：新场景只需符合PG模式并接入通用流水线，无需场景特定工程；（3）快速在线推理：引擎对标准化配置应用统一优化，将PG的开销降至可忽略水平。PG已在淘宝搜索部署，线上A/B实验交易量显著提升+0.47%，GMV提升+0.51%，并已被多个淘宝搜索和推荐团队采用作为生成式检索的迭代框架。

## Abstract
Generative retrieval has become an increasingly adopted paradigm for industrial search, recommendation, and advertising systems, delivering significant online gains. Most existing work combines user behavior sequences with large language models (LLMs) to model user preferences. In practice, feature engineering remains critical to model effectiveness, yet its complexity slows offline iteration and makes online deployment heavy and hard to reuse, all under tight online latency budgets. The root cause is a tight coupling between feature-processing logic and model architecture, where every feature change touches the training and serving code and resists reuse across scenarios. To break this coupling, we present Prompt Generation (PG), a high-level tokenizer and configuration-driven framework that decouples feature-processing logic from model architecture through two declarative JSON files, which serve as the single source of truth for both offline training and online serving, ensuring feature consistency across the two stages. Organizing features under four types with three composable processing components to assemble and compress heterogeneous features, PG delivers acceleration at three levels: (1)fast training iteration: feature experiments require only configuration changes, with built-in token compression for ultra-long sequences; (2)fast deployment: a new scenario only needs to conform to the PG schema and plug into a universal pipeline, with no scenario-specific engineering; (3)fast online inference: engine applies unified optimizations over the standardized configuration, reducing PG's overhead to a negligible level. PG has been deployed on Taobao Search with statistically significant online A/B uplifts of +0.47% in transaction count and +0.51% in GMV, and has been applied across multiple Taobao search and recommendation teams as the iteration framework for generative retrieval.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：生成式检索（Generative Retrieval）在工业搜索、推荐和广告系统中日益普及，但特征工程与模型架构之间存在紧耦合。每次特征变更都需要修改训练和服务代码，导致离线迭代周期长、在线部署沉重且难以跨场景复用，同时受限于严格的在线延迟预算。
- **关键矛盾**：特征工程对模型效果至关重要，但处理逻辑的复杂性严重拖慢了迭代速度，并引发训练-服务不一致（training-serving skew）问题。
- **研究目标**：打破特征处理逻辑与模型架构之间的耦合，设计一个配置驱动的统一框架，实现特征实验的快速迭代、新场景的快速部署以及在线推理的低开销，同时确保离线训练和在线服务的特征一致性。

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想：配置驱动，解耦特征处理

- 通过**两个声明式JSON配置文件**（`prompt_template.json` 和 `prompt_feature.json`）作为单一事实来源（single source of truth），离线训练和在线服务共用同一套配置，确保特征处理逻辑一致。
- 特征处理逻辑与模型架构完全解耦，修改特征只需更改配置，无需修改模型代码或训练/服务流水线。

### 2.2 特征类型分类（四种基本类型）

| 特征类型 | 描述 | 典型用例 |
|---------|------|----------|
| 文本特征（text） | 原始字符串输入，直接由LLM分词嵌入 | 用户查询、商品标题、用户属性 |
| 嵌入特征（emb） | 预计算稠密向量，绕过LLM分词层 | 用户嵌入、商品嵌入 |
| 组合特征（combo） | 组合多个子特征（任意类型），合并为统一表示 | 商品ID+标题+类目 |
| 序列特征（sequence） | 时间/有序集合，每个元素可为上述任意类型 | 点击历史、购买序列 |

### 2.3 三种可组合处理组件

| 组件 | 作用 | 支持的策略/参数 |
|------|------|----------------|
| 预处理器（preprocessor） | 文本标准化（映射、分桶） | `mapping`（字典映射）、`bucketize`（边界分桶） |
| 投影器（projector） | 嵌入维度对齐与适配 | `mlp`（全连接层，可配隐藏维度、激活、Dropout等） |
| 合并器（merger） | 多特征融合，控制输出token数量 | `mean`、`sum`、`concat_mlp`（可学习融合），支持插件扩展 |

### 2.4 关键设计亮点

- **令牌压缩**：通过合并器（如mean-pooling）将多个特征压缩为固定数量token，控制prompt长度，支持超长序列（如45K原始token压缩至11K）。
- **空值处理**：通过`empty_processor`支持填充（fill）或丢弃（drop），确保鲁棒性。
- **前缀/后缀文本**：为特征添加语义标签，仅当特征存在时添加。
- **事件追踪机制**：在线记录原始特征，离线重放，消除训练-服务偏差（达到>99%特征一致性）。

### 2.5 训练与推理架构

- **离线训练**：基于HuggingFace Transformers封装，通过KV-cache机制识别首次前向传播，激活特征处理层替换占位符，后续生成步骤跳过特征处理。
- **在线推理**：分为Prompt Service（特征组织层）和Prompt Generator（计算层），两阶段通信通过`MixedPromptRequest/Response`协议，支持张量复用、批处理、并行流调度等优化。

## 6. 主要结论与发现

1. **没有通用的特征配方**：不同场景最优特征配置不同，PG框架允许低成本探索。
2. **异构特征需要差异化处理**：文本可容忍重压缩，但SID在过度合并（至单个token）时严重降级。
3. **均值合并（mean-pool）是强基线**：在固定token预算下，无参数均值合并通常优于MLP或注意力合并器；学习型合并器应仅在需要内容相关融合时使用。
4. **PG框架有效实现三层面加速**：
   - 快速训练迭代：仅配置变更。
   - 快速部署：即插即用。
   - 快速推理：优化后PG开销占训练步长仅3.2%~15.1%，推理延迟降低40%以上。
5. **线上验证**：
   - 淘宝搜索：交易量+0.47%，GMV+0.51%（14天A/B测试）。
   - 淘宝推荐（ND）：IPV+0.66%，PVR+7.93%（12天测试）。

6. **对齐实验**：Pre-SFT对齐可提升长尾覆盖，HR@5000提升达3%-4.5%。

## 7. 优点

## 8. 不足与局限

- **领域局限**：所有实验均在电商场景（搜索/推荐）进行，未验证在内容推荐、广告、问答等其他领域的通用性。
