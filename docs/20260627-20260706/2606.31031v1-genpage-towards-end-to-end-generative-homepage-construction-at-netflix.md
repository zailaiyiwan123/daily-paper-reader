## 摘要
我们提出了GenPage，一种用于Netflix首页构建的端到端生成式方法，它用单个transformer取代了传统的多阶段推荐器堆栈。GenPage将用户和请求上下文视为提示，并自回归地生成整个结构化的多行首页作为响应。我们调整了LLM训练范式：在生产页面上进行预训练，随后通过加权二元分类（WBC）或强化学习（RL）进行后训练。为了工业级部署，我们引入了应对冷启动、模型更新、业务规则执行和服务效率的技术。在与成熟且高度优化的生产环境首页推荐器进行的在线A/B测试中，GenPage的WBC变体在我们用于发布决策的核心用户参与度指标上提升了+0.24%（p < 0.001），同时将端到端服务延迟降低了20%。离线方面，有两个发现值得注意：在当前状态下，丰富提示带来的改进大于扩展模型容量，并且RL后训练增加了首页多样性，尽管多样性并非目标的一部分。

## Abstract
We present GenPage, an end-to-end generative approach to Netflix homepage construction that replaces the traditional multi-stage recommender stack with a single transformer. GenPage treats the user and request context as a prompt, and autoregressively generates the entire structured, multi-row homepage as the response. We adapt the LLM training recipe: pretraining on production pages, followed by post-training via weighted binary classification (WBC) or reinforcement learning (RL). For industry-scale deployment, we introduce techniques addressing cold start, model freshness, business-rule enforcement, and serving efficiency. In online A/B tests against a mature, highly optimized production homepage recommender, the WBC variant of GenPage delivered a +0.24% lift on the core user engagement metric we use for launch decisions (p < 0.001), while reducing end-to-end serving latency by 20%. Offline, two findings stand out: enriching the prompt yields a larger improvement than scaling model capacity in our current regime, and RL post-training increases homepage diversity even though diversity is not part of the objective.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：传统推荐系统（如Netflix首页）采用多阶段流水线（候选生成、排序、重排），各阶段独立优化，目标不一致；同时需处理结构化多行布局（行和行的实体），导致系统复杂、特征工程繁重、难以端到端优化。此前生成式推荐工作仅关注平面列表（flat list），未涉及结构化页面。
- **研究动机**：受LLM范式启发，探索用一个生成式transformer直接替代整个多阶段堆栈，将用户上下文作为提示（prompt），自回归生成整个结构化首页，以实现端到端建模、更好的扩展性、全页优化（通过RL）以及灵活性和可扩展性。
- **整体含义**：这项工作挑战了工业级推荐系统的传统架构，展示了生成式方法在真实大规模生产环境中的可行性，为未来推荐系统与LLM的融合提供了范例。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：将Netflix首页构建视为序列建模问题。用户上下文（历史、画像、请求）编码为提示（context tokens），整个首页（多行、每行多个实体）编码为响应（page tokens），模型自回归逐token生成整个页面。
- **关键技术细节**：
  - **自定义分词化（Custom tokenization）**：为领域设计专用tokenizer，将实体ID、行ID、用户行为等压缩为单个token，大幅缩短序列长度，降低推理成本；并提供产品控制能力（如业务规则直接对应token级掩码）。
  - **模型架构**：标准decoder-only transformer，解耦输入嵌入和输出投影权重（untied weights）以适应不同训练目标。
  - **训练配方**：
    - **预训练（Next-token prediction）**：在历史正反馈页面数据上训练模型学习“首页语言”。
    - **后训练**：
      - **加权二元分类（WBC）**：将每个实体/行的奖励映射为二元标签（符号）和权重（幅度），优化加权二元交叉熵损失。模型在推理时选择logit最大的token。引入随机负样本（rare tokens作为负例）注入轻度悲观性。
      - **强化学习（RL）**：采用Dr. GRPO算法，训练一个奖励模型（从预训练checkpoint初始化）预测整页奖励（基于内部奖励系统的实体级奖励之和），配合KL散度惩罚防止奖励黑客。支持全页优化、测试时推理、多token实体。
  - **冷启动**：上下文注入新实体元数据；语义嵌入融合（实体ID嵌入 + 内容嵌入）；训练中随机替换实体ID为fallback token。
  - **多节奏增量训练**：周期性大规模训练（大时间窗口） + 每日增量后训练（新数据 + 采样历史数据），使用fallback token处理新token。
  - **业务规则执行**：推理时约束解码，根据规则生成token掩码。
  - **混合行解码**：每行前几个实体自回归生成，后续实体通过单次前向传播选取top-k，兼顾条件依赖和效率。
  - **分页推荐**：增量生成行时，将之前推荐的行和最新用户反馈追加到prompt。
